# OpenCode：最懂程序员的开源 AI 编程助手

最近我把 AI 编程工具全换成了 OpenCode。理由就三个：

1. 完全开源，代码随时可查
2. 厂商中立，不被任何 AI 提供商绑架
3. TUI 体验做到极致，终端党的福音

## 开源不是加分项，是基础款

Claude Code 确实好用，但它是闭源的。你根本不知道它后台干了什么，也不知道哪天会突然收费或者改策略。这种不确定性，对于把代码安全看得比命重的程序员来说，就是定时炸弹。

OpenCode 从第一天就是 100% 开源的。所有代码都在 GitHub 上，社区随时审查、随时贡献。即使开发者明天跑路了，这个项目也能靠社区活下去。

这种透明度带来的安全感，才是开发者工具该有的样子。你的代码、你的数据、你的工作流，永远掌握在自己手里。

## 厂商中立才是未来

OpenCode 最大的杀手锏，是它不绑定任何 AI 提供商。

你想用 Claude 就用 Claude，想用 GPT 就用 GPT，想用 Google 就用 Google，甚至可以在本地跑开源模型。这种灵活性意味着什么？意味着你可以永远用当下性价比最高的方案。

AI 模型价格战打了一年，Claude 降价、GPT 跟进、本地模型越来越强。如果你的工具绑定某个厂商，你只能被动接受。但 OpenCode 让你随时切换，永远站在成本洼地。

这就是厂商中立的价值。你不会因为某个厂商的策略变化就被动挨打，你永远有选择权。

## 终端体验做到变态级别

OpenCode 是由 neovim 用户和 terminal.shop 的开发者做的。他们对终端体验的要求，简直到了变态的地步。

整个工具完全基于 TUI 构建，不是那种给 CLI 套层皮的 Web 应用。键盘操作极其流畅，没有任何鼠标依赖，所有操作都能用快捷键完成。

更骚的是，它内置了两个 Agent，按 Tab 键就能一键切换。

build Agent 是默认的全权限模式，用于写代码、改文件、跑命令，火力全开。

plan Agent 是只读模式，用于分析代码库、理解项目结构、规划改动。你想误删文件都删不了。

这种双 Agent 设计太聪明了。你在探索陌生代码库的时候，用 plan Agent，只看不动，安全第一；你要开始干活了，一键切到 build Agent，该删删该改改，效率拉满。

## 客户端/服务器分离架构

OpenCode 采用客户端/服务器分离架构。这意味着什么？意味着服务端可以跑在你的高性能电脑上，你可以用手机、平板、任何终端设备远程控制它。

这不再是传统的单体应用，而是一个真正的分布式系统。想象一下，你在地铁上用手机查看代码库、规划改动，回到办公室一键切换到 build Agent 开始干活。这种工作流的灵活性，是传统工具做不到的。

TUI 前端只是一个客户端。未来可以有手机客户端、Web 客户端、桌面客户端。核心逻辑都在服务端，所有客户端共享同一个上下文。这才是 AI 编程助手该有的架构。

## 安装方式多到离谱

OpenCode 的安装方式极其丰富，基本上你用任何操作系统、任何包管理器都能一键安装：

```bash
# 一行脚本安装（最省事）
curl -fsSL https://opencode.ai/install | bash

# macOS 和 Linux（推荐，永远最新版）
brew install anomalyco/tap/opencode

# npm/bun/pnpm/yarn
npm i -g opencode-ai@latest

# Windows Scoop
scoop install opencode

# Windows Chocolatey
choco install opencode

# Arch Linux
paru -S opencode-bin

# 任何支持 mise 的系统
mise use -g opencode
```

安装完就能用，不需要配置 API Key，不需要搞复杂的设置。这种开箱即用的体验，才是开发者工具该有的样子。

还有桌面应用版本，支持 macOS、Windows、Linux，全平台覆盖。不想用终端的，直接装桌面版，体验一样的丝滑。

## 内置 LSP 支持

这点很重要。OpenCode 原生支持 LSP（Language Server Protocol），这意味着它能直接读取你项目里的 LSP 配置，理解你代码的语义。

你用 TypeScript、Rust、Go，无论什么语言，OpenCode 都能通过 LSP 准确理解你的代码结构、跳转定义、查找引用。这比那些靠正则匹配和 token 分析的工具要精准太多了。

LSP 支持不是可选项，是必选项。如果一个 AI 编程助手连 LSP 都不支持，它根本不配叫编程工具。

## 总结

OpenCode 不是来替代 Claude Code 的，它是来重新定义 AI 编程助手的。

完全开源、厂商中立、TUI 体验极致、客户端/服务器分离架构、内置 LSP。这些特性组合起来，让它成为了一个真正属于开发者的工具。

如果你也在找一款不会被厂商绑架、体验丝滑、功能强大的 AI 编程助手，OpenCode 值得一试。

更重要的是，它是开源的。即使开发者跑路了，社区也能继续维护下去。这种安全感，才是最核心的竞争力。

AI 编程助手是未来的基础设施。这种基础设施，就必须是开源的、厂商中立的、社区驱动的。OpenCode 走对了路。
